# bee-cache

## 目標

* 多服務架構下，依舊使用本地緩存
* 減少第三方緩存服務的壓力
* 避免網路 __I/O__ 和資料結構轉換的消耗，提高效能
* 相容 __Spring Cacheable__


## 設計

由於緩存通常應用在不頻繁異動的資料上，所以不應該每次都向第三方緩存服務取得資料，而是透過 __MQ Service__ 廣播和訂閱訊息功能，發布註銷的事件，將對應的資料清除，大幅提高系統效能，降低龐大架構的維護成本
